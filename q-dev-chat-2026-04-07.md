## Core Concepts You Need to Learn

**1. The AAA Pattern you already use:**
- **Arrange** → set up preconditions (patient data, system state)
- **Act** → perform the action (click, type, navigate)
- **Assert** → verify the result

This is universal. Every test framework uses this, so you already have the most important mental model.

**2. The Page Object Model (POM)**
This is likely what your framework's `Do()` class is abstracting over. The idea: each "screen" or "page" of the UI gets its own class that encapsulates all interactions with that screen.

```python
# page object - knows HOW to interact with a screen
class PatientEntryPage:
    def __init__(self, driver):
        self.driver = driver
        self.name_field = "patient_name_input"  # locator

    def enter_name(self, name):
        self.driver.find(self.name_field).type(name)

# test - knows WHAT to test
def test_add_patient():
    page = PatientEntryPage(driver)       # Arrange
    page.enter_name("John Doe")           # Act
    assert page.get_confirmation_text()   # Assert
```

**3. Key building blocks of any automation framework:**
- **Driver/Controller** → talks to the UI (Selenium, Appium, pyautogui, or a proprietary SDK)
- **Locators** → how you find UI elements (IDs, XPaths, image recognition, etc.)
- **Page Objects** → classes that wrap screens/pages
- **Actions/Steps** → reusable operations (your `Do()` class is this)
- **Fixtures** → setup/teardown (pytest fixtures handle this)
- **Config** → environment settings, timeouts, device info
- **Reporting** → test results, logs, screenshots on failure

from keyword_adapters.login_adapter import LoginData
